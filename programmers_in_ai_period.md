# 프로그래머의 현재 

- 과도기

# 과도기

- 아직 AI 관련 기술 및 산업은 충분히 성숙하지 못함

- 과거의 예 = 객체 지향 프로그래밍
C++ : 1983년...
SOLID 원칙 : 1980년대부터 논의 되었으나 원칙이 굳어진 것은 2000년대 초

- 현재는 무분별하게 남/오용 되는 여지도 크지 않을까

---

# 프로그래머의 미래

- 혼돈은 언젠가 걷힐 것

- 추상화를 설계하는 직업

- 필경사가 사무직으로 남아있는 것처럼, 장인이 프로그래머로 남아왔던 것처럼 "방식"만 바뀔뿐

- 그 조차도 뺏긴다면.. 이미 직업만의 문제는 아닐것

---

# 결론

- 프로그래머는 원래부터 추상화를 다루던 직업

- AI의 발달로 좀 더 본질적인 추상화를 다루게 됨

- 현재는 개발자도 미숙 + 비개발자들의 진입 = 혼돈

- 방법론이 정립되면 결국 그걸 다루는 것은 프로그래머의 몫
